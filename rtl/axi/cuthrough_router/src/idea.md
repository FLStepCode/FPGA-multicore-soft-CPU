В контексте N действующих независимо алгоритмов - объединить буферы в единое пространство звучит логичной идеей.
Для этого была разработана очередь с индикатором заполненности наполовину.

Теперь возникает вопрос: как со стороны отправителя понять есть ли для пакета место в принимающем узле.

Идея 1: сделать алгоритм, приоритизирующий n-ный алгоритм и n-ный выход. То есть отправляет из первого по номеру
(относительно текущего указателя) пакет в узел, на котором стоит. For (выход) { for (узел) }.

Таким образом удастся в случае дедлока на n-ом канале отправить пакет, котрый был бы изначально отправлен в заблокированный
канал в альтернативных, когда итератор подойдет ближе к альтернативному, нежели заблокированному каналу.
